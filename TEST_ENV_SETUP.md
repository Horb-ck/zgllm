# 测试分支独立运行说明

这份说明只针对当前测试目录，不会影响线上正在运行的 `main` 项目。

## 我做了什么

1. 把测试副本的 Flask 端口从 `7777` 改成了 `7787`
2. 把测试副本的 MCP 端口从 `7778` 改成了 `7788`
3. 关闭了 Flask 的自动重载，避免 `app.py` 启动时把 `mcp_server.py` 拉起两次
4. 把 `mcp_server.py` 里 `FastMCP` 的初始化参数从 `description` 改成了 `instructions`
   当前机器安装的 `mcp` 版本不接受 `description`，不改会在启动时报错

## 为什么这样配

- 线上主项目已经占用 `7777/7778`
- 当前测试目录如果继续使用这两个端口，会和线上服务冲突
- 改成 `7787/7788` 后，测试副本和线上主项目可以同时存在
- 关闭 `use_reloader` 是为了避免 Flask 重载机制导致 MCP 服务重复绑定端口

## 当前测试环境使用的端口

- Flask: `7787`
- MCP: `7788`

## 启动方法

在当前测试目录执行：

```bash
python3 -u app.py
```

如果启动成功，页面入口通常是：

```bash
http://127.0.0.1:7787
```

如果你是远程服务器环境，也可以使用服务器 IP：

```bash
http://<server-ip>:7787
```

## 已知情况

- 如果项目里仍然缺少 `KG` 目录，知识图谱相关功能可能不可用
- 这不会影响你先做大部分前端页面
- 邮件验证码、Canvas 等功能是否可用，取决于 `config.py` 里的真实密钥是否已配置
