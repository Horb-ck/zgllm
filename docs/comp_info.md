# 比赛官网抓取与下载规则表

## 文档说明

本文档用于约束 `Robocon-主赛` 官网监控功能中的内容分类、下载策略、存储策略和后续差异分析入口。

后续开发应以本文件为准，避免将所有官网动态一律下载，导致噪音过多、存储混乱、版本管理失焦。

---

## 1. 总体原则

比赛官网抓取内容分为两条处理链路：

1. **规则资产链路**
   用于处理对比赛方案有直接约束作用的文档，例如规则书、图册、FAQ、补充规则。
   这类内容需要进入下载、版本留存、差异分析、知识库同步流程。

2. **通知动态链路**
   用于处理普通通知、报名说明、赛程公告、中期检查通知等信息。
   这类内容只做消息记录和必要展示，默认不进入附件下载和版本 diff 流程。

---

## 2. 一级分类规则

官网动态抓取后，首先需要对每条消息进行一级分类。

### 2.1 分类定义

| 分类编码 | 分类名称 | 说明 | 是否下载附件 | 是否做版本管理 | 是否进入后续 diff |
| --- | --- | --- | --- | --- | --- |
| `rule_core` | 核心规则文档 | 规则书、竞技赛规则、正式补充规则 | 是 | 是 | 是 |
| `figure_core` | 核心图册文档 | 图册、场地图、结构图、尺寸图 | 是 | 是 | 是 |
| `faq_core` | 核心答疑文档 | FAQ、官方答疑、裁判补充说明 | 是 | 是 | 是 |
| `important_notice` | 重要通知 | 对比赛安排有重要影响的通知，但不属于规则正文 | 否 | 否 | 否 |
| `general_notice` | 普通通知 | 一般动态、报名、会议、名单、公示等 | 否 | 否 | 否 |

---

## 3. 标题分类规则

### 3.1 自动归类为 `rule_core`

如果标题包含以下关键词之一，优先归类为 `rule_core`：

- `竞技赛规则`
- `规则书`
- `比赛规则`
- `规则V`
- `补充规则`
- `规则修订`

说明：

- 这类文档通常是比赛设计和项目决策的核心依据。
- 进入下载、版本留存和差异分析链路。

### 3.2 自动归类为 `figure_core`

如果标题包含以下关键词之一，优先归类为 `figure_core`：

- `图册`
- `场地图`
- `尺寸图`
- `结构图`
- `附件图`

说明：

- 这类内容通常与机构尺寸、布局、路径规划等直接相关。
- 进入下载、版本留存和差异分析链路。

### 3.3 自动归类为 `faq_core`

如果标题包含以下关键词之一，优先归类为 `faq_core`：

- `FAQ`
- `答疑`
- `补充说明`
- `裁判说明`
- `问题解答`

说明：

- 这类内容经常会对规则正文作补充甚至修正。
- 进入下载、版本留存和差异分析链路。

### 3.4 自动归类为 `important_notice`

如果标题包含以下关键词之一，默认归类为 `important_notice`：

- `赛程`
- `时间安排`
- `中期检查`
- `技术交流`
- `测试安排`
- `参赛说明`
- `现场说明`

说明：

- 这类内容对比赛准备有影响，但一般不属于规则正文。
- 默认只记录，不下载。

### 3.5 自动归类为 `general_notice`

如果标题包含以下关键词之一，默认归类为 `general_notice`：

- `通知`
- `报名`
- `公示`
- `名单`
- `会议`
- `举办`
- `资格`
- `结果`
- `章程`

说明：

- 这类内容通常不直接进入规则版本链路。
- 默认只保留标题、日期、链接和必要摘要。

---

## 4. 下载规则

### 4.1 必须自动下载

以下分类的内容必须自动下载附件：

- `rule_core`
- `figure_core`
- `faq_core`

下载内容包括：

- PDF
- DOC / DOCX
- XLS / XLSX
- ZIP / RAR

前提是附件确实属于规则资产而不是辅助表单。

### 4.2 默认不下载

以下分类默认不下载附件：

- `important_notice`
- `general_notice`

这些内容只记录：

- 标题
- 日期
- 详情页链接
- 来源
- 页面摘要

### 4.3 附件级二次过滤

即使一条消息已进入可下载类别，也不是详情页里的所有附件都下载。

只有附件名命中以下关键词时，才优先下载：

- `规则`
- `rule`
- `图册`
- `faq`
- `答疑`
- `武林探秘`
- `补充说明`

以下附件名命中时，默认不下载：

- `报名表`
- `回执`
- `盖章`
- `汇总表`
- `名单`
- `签到`
- `申请表`
- `说明会`

---

## 5. 存储规则

### 5.1 规则资产链路存储内容

对于 `rule_core`、`figure_core`、`faq_core`，需要保存：

1. 消息元数据
2. 附件元数据
3. 原始文件本地副本
4. 可分析文本
5. 版本记录
6. 更新事件

### 5.2 通知动态链路存储内容

对于 `important_notice`、`general_notice`，默认只保存：

1. 标题
2. 日期
3. 详情页 URL
4. 来源页面
5. 页面摘要
6. 抓取时间

说明：

- 除非后续人工升级分类，否则不下载原始附件。

---

## 6. 进入版本管理的条件

只有下列分类进入版本管理：

- `rule_core`
- `figure_core`
- `faq_core`

版本管理至少需要记录：

- 文档名称
- 比赛归属
- 发布时间
- 来源 URL
- 文件 hash
- 文本 hash
- 本地文件路径
- 文本路径
- 是否为最新版本

---

## 7. 进入差异分析的条件

只有下列分类进入差异分析：

- `rule_core`
- `figure_core`
- `faq_core`

并且必须满足：

1. 已存在上一版本
2. 本次文件 hash 或文本 hash 发生变化

不进入差异分析的情况：

- 只是普通通知
- 没有上一版本
- 只是页面格式变化且正文无变化

---

## 8. 知识库同步规则

### 8.1 需要同步到知识库的内容

以下内容建议同步到 FastGPT：

- 最新规则全文
- 历史规则版本全文
- 图册文本
- FAQ 文本
- 差异分析摘要

### 8.2 不优先同步的内容

以下内容默认不优先同步到知识库：

- 一般通知
- 报名信息
- 名单公示
- 会务信息

说明：

- 这些内容更多用于平台展示，不一定需要进入问答检索。

---

## 9. 当前推荐的执行标准

### 9.1 自动下载标准

当前阶段，只有标题和附件命中下列语义时，才进入自动下载：

- `竞技赛规则`
- `比赛规则`
- `规则书`
- `规则V`
- `图册`
- `FAQ`
- `答疑`
- `补充规则`
- `补充说明`

### 9.2 只记录不下载标准

当前阶段，下列语义的动态只做消息记录：

- `通知`
- `报名`
- `中期检查`
- `赛程`
- `会议`
- `公示`
- `名单`
- `举办`
- `资格`

---

## 10. 后续开发约束

后续代码实现时，应遵守以下约束：

1. 不允许将所有官网条目一律进入下载链路。
2. 不允许将所有附件一律下载。
3. 规则资产与通知动态必须分开处理。
4. 只有规则资产才进入版本管理和差异分析。
5. 知识库应以规则资产和分析结果为主，不应被大量普通通知污染。

---

## 11. 当前结论

当前系统对 Robocon 官网内容的处理标准定义为：

- **规则 / 图册 / FAQ / 补充规则**：下载、留存、版本管理、差异分析、知识库同步。
- **普通通知 / 报名 / 公示 / 会议 / 名单等**：只记录，不下载，不做版本 diff。

后续围绕 `Robocon-主赛` 官网抓取逻辑的修改，均以本规则表为基准执行。
